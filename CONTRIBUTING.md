# Contributing Guide

Thank you for your interest in the Awesome Agent Protocol project! We welcome contributions of any kind.

## How to Contribute

### Adding New Projects

1. Ensure the project meets the following criteria:
   - Related to Agent protocols, MCP, A2A, or ACP
   - Is open source or has public documentation
   - Has a certain level of quality and activity
   - Not already in the list

2. Add the project in the following format:
   ```markdown
   - [Project Name](Project URL) - Brief description
   ```

3. Add the project to the appropriate category and subcategory

4. Sort alphabetically (optional but recommended)

### Modifying Existing Content

- Fix errors or update links
- Improve descriptions
- Adjust categories

## PR Submission Process

1. Fork this repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add some amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Formatting Requirements

- Use Markdown format
- Link format: `[Project Name](URL)`
- Keep descriptions concise, under 100 characters
- Maintain alphabetical order of lists
- One project per line

## Category Descriptions

### MCP (Model Context Protocol)
- **Official Protocols**: MCP official specifications and documentation
- **SDKs/Libraries**: MCP development toolkits for various languages
- **Tools**: MCP-related tools
- **Applications**: Applications with MCP support

### A2A (Agent-to-Agent)
- **Collaboration Protocols**: Agent-to-Agent communication protocols
- **Multi-Agent Frameworks**: Multi-agent development frameworks
- **Coordination Tools**: Coordination and monitoring tools

### ACP (Agent Client Protocol)
- **Client Protocols**: Client API protocols
- **Desktop Apps**: Desktop client applications
- **Web Apps**: Web applications
- **Mobile Apps**: Mobile applications

## Code of Conduct

Please respect other contributors and maintain friendly and professional communication.

## Questions

Have any questions? Please open an Issue to discuss.

Thank you again for your contribution!
