# 贡献指南

感谢你对 Awesome Agent Protocol 项目感兴趣！我们欢迎任何形式的贡献。

## 如何贡献

### 添加新项目

1. 确保项目符合以下标准：
   - 与 Agent 协议、MCP、A2A 或 ACP 相关
   - 是开源项目或有公开文档
   - 有一定的质量和活跃度
   - 不在列表中

2. 按照以下格式添加项目：
   ```markdown
   - [项目名称](项目链接) - 简短描述
   ```

3. 将项目添加到合适的分类和子分类中

4. 按字母顺序排序（可选但推荐）

### 修改现有内容

- 修正错误或更新链接
- 改进描述
- 调整分类

## 提交 PR 的流程

1. Fork 本仓库
2. 创建一个新的分支 (`git checkout -b feature/amazing-feature`)
3. 提交你的修改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 开启一个 Pull Request

## 格式要求

- 使用 Markdown 格式
- 链接格式：`[项目名称](URL)`
- 描述简洁明了，不超过 100 个字符
- 保持列表的字母顺序
- 每个项目占一行

## 分类说明

### MCP (Model Context Protocol)
- **官方协议**：MCP 官方规范和文档
- **SDK/库**：各语言的 MCP 开发工具包
- **工具**：MCP 相关工具
- **应用**：支持 MCP 的应用

### A2A (Agent-to-Agent)
- **协作协议**：Agent间通信协议
- **多Agent框架**：多Agent开发框架
- **协调工具**：协调和监控工具

### ACP (Agent Client Protocol)
- **客户端协议**：客户端API协议
- **桌面应用**：桌面客户端应用
- **Web应用**：Web应用
- **移动应用**：移动应用
